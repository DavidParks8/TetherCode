## App Store Connect — Notes for Review

```
Clawdex is a companion app for a bridge that the user runs on their own
infrastructure. The app connects to that bridge over a private network and lets
the user drive coding-agent runs, review approvals/Git/diffs, and run allowlisted
terminal commands on the host they control. It is not a public multi-tenant shell.

A dedicated review bridge is provided on an isolated private network so no
product account creation is needed. The bridge is not exposed to the public

- Support contact during review: [FILL: name + email + phone]

Walkthrough:
1. Enroll in and connect the supplied review VPN/private overlay.
2. Launch the app. On "Connect Your Bridge", enter the private bridge URL and
   bridge token above, then tap Test Connection and continue.
3. Start a new run, send a prompt, and confirm a response is received.
4. Open the Git screen to view status/diff.
5. If prompted, review and approve an action.

New in 5.2.3 — Push notifications:
- The bridge sends a push when an agent turn completes or needs approval (the
  app suspends its socket when backgrounded, so the bridge is the sender). Delivery
  path: bridge -> Expo push service -> APNs.
- To exercise it: enable notifications when prompted (or in Settings), background
  the app, then trigger a turn on the review bridge. A notification arrives;
  approval notifications include Approve/Deny actions. Selecting an action
  foregrounds the app and resolves over the matching authenticated bridge socket.
- Expo receives the push token, visible title/body, and event/routing identifiers.
  A completion body may contain a short reply preview (<=140 chars); generic
  completion and approval text contains the review project folder name. No
  prompts, code, diffs, or tool output are sent. Notifications are user-toggleable
  in Settings. Push requires outbound HTTPS only; the bridge remains private.

The review environment uses a sanitized workspace, firewall rules limiting bridge
ports to the VPN/overlay, and separate temporary VPN and bridge credentials. Those
credentials will be revoked and the environment shut down after review.
```

## App Privacy (App Store) / Data Safety (Play) — answers

Declare these for 5.2.3 because, when notifications are enabled, notification
text, identifiers, and a push token transit Expo and the platform push service.

Data types collected:
1. Other user content — a truncated (<=140 char) snippet of the assistant's reply,
   included in turn-completed notifications.
   - Linked to the user's identity: No
   - Used for tracking: No
   - Purpose: App Functionality (delivering user-enabled notifications)
   - Optional: Yes (controlled by the in-app notifications toggle)
2. Device ID / push token — an opaque identifier used solely to route the
   notification to the device.
   - Linked to the user's identity: No
   - Used for tracking: No
   - Purpose: App Functionality
   - Optional: Yes
3. Other identifiers — notification, bridge-profile, registration, thread, and
   approval identifiers used for routing, deep linking, and safe action handling.
   - Linked to the user's identity: No
   - Used for tracking: No
   - Purpose: App Functionality
   - Optional: Yes

No advertising identifiers. No data sold. No data used for tracking across apps.
